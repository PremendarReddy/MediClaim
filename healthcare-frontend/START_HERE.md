- **Demo OTP:** `123456` (when registering patients)

## 🏥 Three User Roles to Explore

### 1️⃣ Hospital Role
**What they do:** Add patients, submit claims, upload medical reports

**Try this:**
1. Login as Hospital
2. Go to Dashboard → Click "Add Patient"
3. Fill patient details
4. Enter OTP: `123456`
5. View patients list
6. Upload medical reports

**Time:** 5 minutes

---

### 2️⃣ Patient Role
**What they do:** Upload documents, view claims, chat with AI, manage insurance

**Try this:**
1. Login as Patient
2. Go to Dashboard
3. View Insurance → Check coverage
4. Go to Claims → Upload documents
5. Go to AI Analysis → Chat with bot
6. View Medical Reports

**Time:** 5 minutes

---

### 3️⃣ Insurance Company Role
**What they do:** Review claims, verify documents, approve claims

**Try this:**
1. Login as Insurance
2. Go to Dashboard → View metrics
3. Go to Claims → Filter by risk
4. View a claim detail
5. Check claim documents
6. Update claim status

**Time:** 5 minutes

## 🌟 Key Features to Try

### ✨ AI Claim Agent
- **Path:** Patient → AI Analysis
- **Features:** Chat with AI, analyze reports, view history
- Ask: "What documents do I need?" or "Check my claim status"

### 📋 Patient Onboarding
- **Path:** Hospital → Add Patient
- **Features:** 3-step wizard with OTP verification
- Demo OTP: `123456`

### 📊 Claims Management
- **Path:** Hospital/Insurance/Patient → Claims
- **Features:** Filter, search, track status, upload documents

### 🏥 Insurance Coverage
- **Path:** Patient → Insurance
- **Features:** View coverage limits, treatment types, remaining balance

## 🐛 Troubleshooting

### App won't start?
```bash
npm install
npm run dev
```

### Data not persisting?
- Check browser allows localStorage
- Clear cache: Ctrl+Shift+Delete
- Refresh page

### Can't login?
- Use any email/password combination
- Make sure to **select a role** (Hospital/Patient/Insurance)

### OTP not working?
- Always use: **123456**
- Make sure you're in the text input field

### Need help?
→ Check **[README_SETUP.md](README_SETUP.md)** → Troubleshooting section
